Q&A

  1. The Three main diffrences between web1,web2 and web3

    1. web1 is read-only. Displays superficial data. There is no interaction between people, users are used to passively receive information. In web2 it is writable, allows interaction between users and sites and among themselves.
      web3 is excutable ie running dynamic applications, here it starts to get information processing and AI integration
    2. In web1 crawlers cannot be used and web2 and web3 have the option to use.
    3. In web1 an internet search can be performed on one specific address. Web2 has the option of searching with defined tags. In web3, the option to go beyond searching in web2 was also added using semantic search.
      Additionally in web1 there is no refreshment at all. In web2 there is a small amount of refreshes and in web3 there is the greatest amount of refreshes.

  2. Medicines side effects Use Case

    3. Let's take an example of a side effect of a specific drug- we take the query: "acamol side effects"

    1. UC Name: Example For web1:
      Primary actor: User
      Main scenario:
      1. Write the specific site that talks about the side effects of acamol.
      2. Click 'Search'.
      Result: site specific
    2. UC Name: Example For web2:
      Primary actor: User
      Main scenario:
      1. Type in the search box: "Acmol Side Effects"
      2. Click 'Search'.
      Result: Links to web pages with the tags of the words in the query.
    3. UC Name: Example For web3:
      Primary actor: User
      Main scenario:
      1. Type in the search box: "Acmol Side Effects"
      2. Click 'Search'.
      Results: Links to web pages that are also semantically related (in addition to what is said in web2) to the query.
